Étape 2 : Propriétés du Sol (Modèle van Genuchten)

Le sol du Gharb (plaine céréalière du Maroc occidental) présente deux horizons distincts. Les propriétés hydrodynamiques sont décrites par le modèle van Genuchten :

$$\theta(h) = \theta_r + \frac{\theta_s - \theta_r}{[1 + |\alpha h|^n]^m}, \quad m = 1 - \frac{1}{n}$$ $$K(\theta) = K_s \left\{\left[\frac{\theta - \theta_r}{\theta_s - \theta_r}\right]^l \left\{1 - \left[1 - \left(\frac{\theta - \theta_r}{\theta_s - \theta_r}\right)^{1/m}\right]^m\right\}^2\right\}$$

Couche 1 (0–40 cm) : Limon argileux

ParamètreNotationValeur
Teneur en eau résiduelle$\theta_r$0.078 cm$^3$/cm$^3$
Teneur en eau saturante$\theta_s$0.43 cm$^3$/cm$^3$
Paramètre d'air-entry$\alpha$0.036 cm$^{-1}$
Indice de porosité$n$1.56
Conductivité saturante$K_s$1.04 cm/j
Paramètre de torsion$l$0.5

Couche 2 (40–100 cm) : Argile

ParamètreNotationValeur

Teneur en eau saturante$\theta_s$0.38 cm$^3$/cm$^3$

Étape 3 : Conditions Initiales

Au début de la simulation, le sol est en état d'équilibre hydrostatique avec une charge matricielle uniforme :

$$h_0 = -200 \text{ cm}$$

Cette valeur correspond à un sol relativement sec (teneur en eau intermédiaire), représentant les conditions après une période sans irrigation.

Attention
La charge $h$ est négative dans la zone non saturée. Une valeur de $-200$ cm signifie un potentiel matriciel de $-200$ cm d'eau, soit environ $-20$ kPa.

Exercice 2 : Analyse des Résultats avec R

Note
Durée estimée : 20 minutes

Étape 1 : Lecture des Fichiers de Sortie HYDRUS

Après la simulation, HYDRUS génère plusieurs fichiers ASCII. Les deux principaux sont :

  • PROFILE.OUT : profils de teneur en eau $\theta$ et charge $h$ aux temps de sortie
  • T_LEVEL.OUT : variables intégrées dans le temps (flux, cum)
Afficher la structure PROFILE.OUT
# [Col  1] Time = 1.000000 [L]
# [Col  2] Depth [L]
# [Col  3] Theta [L3/L3]
# [Col  4] h [L]
  Time   Depth   Theta    h
  1.00   0.00   0.280  -50.5
  1.00   0.01   0.285  -48.2
  ...

Le fichier est organisé par blocs (un bloc par temps de sortie). Code R pour charger les données :

Afficher le code R
# Lecture du fichier PROFILE.OUT
profile_file <- "PROFILE.OUT"

# Fonction pour parser PROFILE.OUT
read_hydrus_profile <- function(filename) {
  lines <- readLines(filename)

  # Identifier les lignes de donnees
  data_lines <- grep("^[^#]", lines)

  # Parser
  data <- read.table(filename, skip=0)
  colnames(data) <- c("Temps", "Profondeur", "Teneur_eau", "Charge")

  # Convertir profondeur en cm
  data$Profondeur_cm <- data$Profondeur * 100

  return(data)
}

# Charger les donnees
profile_data <- read_hydrus_profile(profile_file)

# Verifier les premieres lignes
head(profile_data)
Note
Selon votre version de HYDRUS et le système d'exploitation, le format exact peut varier. Il est possible de lire avec read.fwf() (colonnes de largeur fixe) si les données sont fortement formatées.

Étape 3 : Évolution Temporelle à Profondeurs Spécifiques

Suivre la teneur en eau aux profondeurs 10 cm, 30 cm et 60 cm pour observer la réponse rapide en surface et le drainage en profondeur.

Afficher le code R
# Teneur en eau a profondeurs specifiques
z_interets <- c(10, 30, 60)  # cm

# Pour chaque profondeur, extraire theta et temps
theta_evolution <- data.frame(
  Temps = numeric(),
  Profondeur = numeric(),
  Theta = numeric()
)

for(z in z_interets) {
  indices <- which(abs(profile_data$Profondeur_cm - z) < 0.5)

  if(length(indices) > 0) {
    for(idx in indices) {
      theta_evolution <- rbind(theta_evolution,
        data.frame(
          Temps = profile_data$Temps[idx],
          Profondeur = z,
          Theta = profile_data$Teneur_eau[idx]
        )
      )
    }
  }
}

# Graphique temporel
png("theta_vs_temps.png", width=900, height=600)
plot(NULL, xlim=c(0, 30), ylim=c(0.05, 0.45),
     xlab="Temps (jours)", ylab="Teneur en eau (cm3/cm3)",
     main="Evolution temporelle de la teneur en eau")

# Couleurs par profondeur
col_map <- c("10"="green", "30"="orange", "60"="darkred")

for(z in z_interets) {
  subset_data <- theta_evolution[theta_evolution$Profondeur == z, ]
  subset_data <- subset_data[order(subset_data$Temps), ]
  lines(subset_data$Temps, subset_data$Theta,
        type="o", lwd=2, col=col_map[as.character(z)])
}

# Marquer les jours d'irrigation
abline(v=c(1, 11, 21), col="blue", lty=2, lwd=1.5)
text(c(1, 11, 21), 0.42, "Irrig.", cex=0.8)

legend("bottomright",
       paste("z =", z_interets, "cm"),
       col=as.character(col_map[as.character(z_interets)]),
       lwd=2)
grid()
dev.off()

cat("Graphique sauve : theta_vs_temps.png\n")

Étape 4 : Bilan Hydrique et Efficience de l'Irrigation

Le bilan hydrique cumulé permet d'évaluer l'efficience :

$$\text{Efficience} = \frac{\text{Évapotranspiration réelle (ETa)}}{\text{Irrigation totale}} \times 100$$

HYDRUS fournit dans T_LEVEL.OUT les flux cumulés :

Afficher le code R
# Lecture du fichier T_LEVEL.OUT
t_level_file <- "T_LEVEL.OUT"

# Charger les donnees
t_level <- read.table(t_level_file, header=TRUE, skip=2)

# Cumules sur 30 jours
total_irrigation <- sum(t_level$FluxTop[t_level$FluxTop > 0], na.rm=TRUE)
total_drainage <- abs(sum(t_level$FluxBot[t_level$FluxBot < 0], na.rm=TRUE))
total_ET <- sum(t_level$Transpiration + t_level$Evaporation, na.rm=TRUE)
stockage_final <- t_level$Stockage[nrow(t_level)] -
                  t_level$Stockage[1]

# Efficience
efficience <- total_ET / total_irrigation * 100

cat("=== BILAN HYDRIQUE 30 JOURS ===\n\n")
cat("Irrigation totale :", round(total_irrigation, 1), "mm\n")
cat("Drainage total :", round(total_drainage, 1), "mm\n")
cat("Evapotranspiration :", round(total_ET, 1), "mm\n")
cat("Variation de stockage :", round(stockage_final, 1), "mm\n")
cat("Efficience irrigation :", round(efficience, 1), "%\n")

# Verification du bilan
verification <- total_irrigation - total_drainage - total_ET - stockage_final
cat("Erreur bilan (residuel) :", round(verification, 2), "mm\n")

# Chemins des fichiers de sortie
scenarios <- list(
  A = "Scenario_A/PROFILE.OUT",
  B = "Scenario_B/PROFILE.OUT",
  C = "Scenario_C/PROFILE.OUT"
)

# Fonction pour lire et resumer les donnees
resume_scenario <- function(profile_file) {
  profile <- read_hydrus_profile(profile_file)

  # Extraire la teneur en eau a z=30 cm a tous les temps
  z_ref <- 30
  indices <- which(abs(profile$Profondeur_cm - z_ref) < 0.5)

  data_z30 <- profile[indices, ]
  return(data_z30)
}

# Charger donnees pour tous les scenarios
data_A <- resume_scenario(scenarios$A)
data_B <- resume_scenario(scenarios$B)
data_C <- resume_scenario(scenarios$C)

# Graphique superpose
png("comparaison_scenarios.png", width=900, height=600)
plot(data_A$Temps, data_A$Teneur_eau, type="o",

Étape 5 : Tableau Comparatif

Construire un tableau récapitulatif avec les métriques clés pour chaque scénario :

Afficher le code R
# Fonction pour extraire bilan hydrique d'un scenario
extraire_bilan <- function(t_level_file) {
  t_level <- read.table(t_level_file, header=TRUE, skip=2)

  total_irrig <- sum(t_level$FluxTop[t_level$FluxTop > 0], na.rm=TRUE)
  total_drain <- abs(sum(t_level$FluxBot[t_level$FluxBot < 0], na.rm=TRUE))
  total_ET <- sum(t_level$Transpiration + t_level$Evaporation, na.rm=TRUE)

  efficience <- total_ET / total_irrig * 100

  return(data.frame(
    Irrigation_mm = round(total_irrig, 1),
    Drainage_mm = round(total_drain, 1),
    ET_mm = round(total_ET, 1),
    Efficience_pct = round(efficience, 1)
  ))
}

# Bilans pour les 3 scenarios
bilan_A <- extraire_bilan("Scenario_A/T_LEVEL.OUT")
bilan_B <- extraire_bilan("Scenario_B/T_LEVEL.OUT")
bilan_C <- extraire_bilan("Scenario_C/T_LEVEL.OUT")

# Tableau comparatif
tableau <- rbind(
  data.frame(Scenario = "A (3x20mm)", bilan_A),
  data.frame(Scenario = "B (6x10mm)", bilan_B),
  data.frame(Scenario = "C (30x2mm)", bilan_C)
)

print(tableau)

# Exporter en CSV
write.csv(tableau, "comparaison_bilans.csv", row.names=FALSE)
Attention
Vérifiez que les fichiers T_LEVEL.OUT de chaque scénario contiennent les bonnes colonnes. Selon votre version de HYDRUS et vos paramètres de sortie, les noms de colonnes peuvent varier (par ex. RootUp, EvapSur, Q, etc.).

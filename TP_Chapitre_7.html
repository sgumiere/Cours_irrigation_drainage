TP 7 — Géostatistique et bilan hydro-salin

GAE-3007 · Drainage · Chapitre 7 : Log-normalité de K, variogramme, krigeage, bilan sel (ISAWIP)
Université Laval · Prof. Silvio J. Gumiere · Durée : 1 h · Environnement : R
Objectifs pédagogiques
  • Tester la log-normalité d'un jeu de mesures de \(K\) et calculer les moyennes arithmétique, géométrique et harmonique.
  • Construire un variogramme expérimental et ajuster un modèle (sphérique, exponentiel).
  • Réaliser un krigeage ordinaire et cartographier la carte d'écartement \(L(x,y)\).
  • Établir un bilan hydrique et salin (leaching fraction, sel exporté) pour un projet type ISAWIP.
  • Évaluer la fraction de lessivage nécessaire pour maintenir la salinité racinaire sous un seuil critique.
Prérequis R 
library(tidyverse)
library(gstat)
library(sf)
Équations clés

Log-normalité : si \(\ln K \sim \mathcal{N}(\mu, \sigma^2)\), alors \(\bar K_{geom} = e^{\mu}\) et \(\bar K_{arith} = e^{\mu + \sigma^2/2}\).

Variogramme expérimental :

\[ \gamma(h) = \frac{1}{2\,N(h)}\sum_{i=1}^{N(h)} [Z(x_i) - Z(x_i+h)]^2 \]

Modèle sphérique :

\[ \gamma(h) = c_0 + c\left[\tfrac{3h}{2a} - \tfrac{1}{2}\left(\tfrac{h}{a}\right)^3\right] \text{ si } h \le a \]

Bilan salin (steady state) :

\[ EC_{drain} = \frac{EC_{irr}}{LF}, \qquad LF = \frac{V_{drain}}{V_{irr}+V_{pluie}} \]

Leaching requirement (Rhoades) : \(LF_{req} = \dfrac{EC_{irr}}{5\cdot EC_{seuil} - EC_{irr}}\).

Exercice 1 — Log-normalité de K et moyennes effectives 12 min

Un jeu de 80 mesures de \(K\) (m/j) provient d'un projet ISAWIP en Égypte.

set.seed(1)
K_obs <- exp(rnorm(80, mean = log(0.6), sd = 0.9))   # m/j
  1. Tracez l'histogramme de \(K\) et de \(\ln K\). Réalisez un test de Shapiro sur \(\ln K\).
  2. Calculez \(\bar K_{arith}\), \(\bar K_{geom}\), \(\bar K_{harm}\) et les reliez \(\sigma_{\ln K}\).
  3. Laquelle est pertinente pour : (a) un écoulement parallèle horizontal ; (b) en série vertical ; (c) la moyenne spatiale d'une variable log-normale ?
Solution 
set.seed(1)
K_obs <- exp(rnorm(80, mean = log(0.6), sd = 0.9))

# stats
K_arith <- mean(K_obs)
K_geom  <- exp(mean(log(K_obs)))
K_harm  <- 1 / mean(1/K_obs)
sig_ln  <- sd(log(K_obs))

cat(sprintf("arith = %.3f | geom = %.3f | harm = %.3f | σ_lnK = %.2f\n",
            K_arith, K_geom, K_harm, sig_ln))

shapiro.test(log(K_obs))        # test log-normalité

# relation théorique : arith = geom * exp(σ²/2)
cat("geom * exp(σ²/2) =", K_geom * exp(sig_ln^2/2), "≈ arith\n")

par(mfrow = c(1,2))
hist(K_obs, main = "K (m/j)", xlab = "K")
hist(log(K_obs), main = "ln K", xlab = "ln K")

Ordre attendu : \(\bar K_{arith} > \bar K_{geom} > \bar K_{harm}\) ; le test Shapiro sur \(\ln K\) ne rejette pas \(H_0\) (log-normalité admise).

(a) écoulement parallèle → arithmétique ; (b) série vertical → harmonique ; (c) moyenne spatiale log-normale → géométrique (c'est celle à utiliser pour les équations de drainage type Hooghoudt).

Exercice 2 — Variogramme expérimental et modèle sphérique 15 min

On dispose de 60 mesures de \(\ln K\) avec coordonnées (x, y) en m.

set.seed(7)
n   <- 60
pts <- tibble(x = runif(n, 0, 500),
              y = runif(n, 0, 500))
# tendance spatiale légère + variabilité log-normale
pts <- pts |>
  mutate(lnK = 0.002*x - 0.003*y + rnorm(n, 0, 0.9))
  1. Convertissez en objet sf et construisez le variogramme expérimental (gstat::variogram).
  2. Ajustez un modèle sphérique (fit.variogram).
  3. Interprétez les paramètres : pépite \(c_0\), seuil \(c_0 + c\), portée \(a\).
Solution 
library(sf); library(gstat)
set.seed(7)
n <- 60
pts <- tibble(x = runif(n, 0, 500),
              y = runif(n, 0, 500)) |>
  mutate(lnK = 0.002*x - 0.003*y + rnorm(n, 0, 0.9)) |>
  st_as_sf(coords = c("x","y"))

vgm_exp <- variogram(lnK ~ 1, pts)
vgm_fit <- fit.variogram(vgm_exp, model = vgm(psill = 0.7,
                                              model  = "Sph",
                                              range  = 200,
                                              nugget = 0.2))
print(vgm_fit)

plot(vgm_exp, vgm_fit)

Interprétation :

  • Pépite \(c_0\) : variance aux très courtes distances — erreurs de mesure + micro-variabilité.
  • Portée \(a\) : distance au-delà de laquelle les points sont spatialement indépendants (\(\approx 200\) m ici).
  • Seuil \(c_0+c\) : variance totale du champ (doit \(\approx \sigma^2_{\ln K}\)).

Si \(a < \) largeur du champ, un design unique par parcelle est acceptable ; sinon, il faut une cartographie + écartements variables.

Exercice 3 — Krigeage et carte d'écartement L(x,y) 15 min

À partir des \(\ln K\) de l'exercice 2, construisez une carte de \(K(x,y)\) puis \(L(x,y)\) par Hooghoudt simplifié (\(d=3\) m, \(h=0{,}3\) m, \(q=0{,}012\) m/j).

Solution 
# grille régulière
grd <- expand_grid(x = seq(0, 500, 10), y = seq(0, 500, 10)) |>
  st_as_sf(coords = c("x","y"))

krig <- krige(lnK ~ 1, pts, grd, model = vgm_fit)

# transformation inverse : K = exp(lnK)
d <- 3; h <- 0.3; q <- 0.012
carte <- krig |>
  mutate(K = exp(var1.pred),
         L = sqrt((4*K*h^2 + 8*K*d*h)/q))

st_coords <- st_coordinates(carte)
carte_df <- tibble(x = st_coords[,1], y = st_coords[,2],
                   L = carte$L)

ggplot(carte_df, aes(x, y, fill = L)) +
  geom_raster() +
  scale_fill_viridis_c(name = "L (m)") +
  coord_equal() +
  labs(title = "Carte d'écartement drainé L(x,y)") + theme_minimal()

# classement par zones de design
carte_df <- carte_df |>
  mutate(zone = cut(L, breaks = c(0, 15, 20, 25, 100),
                    labels = c("<15 m","15–20","20–25",">25")))
table(carte_df$zone)

Lecture : si la carte montre 2–3 zones dominantes, on conçoit 2–3 sous-parcelles avec écartements distincts (drainage « à la carte »). Cette approche peut économiser 10–25 % du linéaire total sur des champs hétérogènes.

Exercice 4 — Bilan hydrique et fraction de lessivage 10 min

Contexte ISAWIP (Égypte) : irrigation annuelle 1400 mm, pluie 25 mm, ETc 1200 mm, \(EC_{irr}=1{,}2\) dS/m, \(EC_{seuil}\) (luzerne) = 2 dS/m.

  1. Volume drainé = Irr + P − ETc. Calculez la fraction de lessivage effective \(LF = V_{drain}/(V_{irr}+V_{pluie})\).
  2. \(EC_{drain}\) estimée à l'équilibre.
  3. \(LF_{req}\) (Rhoades). Conclure : le drainage suffit-il à maintenir la salinité sous le seuil ?
Solution 
Irr  <- 1400
P    <- 25
ETc  <- 1200
Vdra <- Irr + P - ETc      # = 225 mm
LF   <- Vdra / (Irr + P)
EC_i <- 1.2
EC_s <- 2.0

EC_drain  <- EC_i / LF
LF_req    <- EC_i / (5*EC_s - EC_i)

cat(sprintf("V_drain = %.0f mm ; LF = %.2f\n", Vdra, LF))
cat(sprintf("EC_drain = %.2f dS/m\n", EC_drain))
cat(sprintf("LF_req (Rhoades) = %.2f\n", LF_req))
cat(ifelse(LF >= LF_req,
           "OK : drainage suffisant pour le lessivage.\n",
           "Insuffisant : augmenter l'irrigation ou réduire les cultures sensibles.\n"))

Résultats : \(V_{drain}=225\) mm, \(LF\approx 0{,}16\), \(EC_{drain}\approx 7{,}5\) dS/m (nappe très salée — exportée par drains), \(LF_{req}\approx 0{,}14\). Comme \(LF > LF_{req}\) : le drainage assure l'objectif (avec marge faible). Toute baisse de l'irrigation rendrait le bilan salin défavorable.

Exercice 5 — Export de sel annuel et scénarios CC 10 min

Prolongez le bilan sur 3 scénarios climat : base, CC+1 (ETc +10 %, pluie −10 %), CC+2 (ETc +20 %, pluie −20 %).

Solution 
# 1 dS/m ≈ 0.64 g/L de sels totaux
f_EC_TDS <- 0.64

scen <- tibble(
  scenario = c("base","CC+1","CC+2"),
  Irr      = c(1400, 1400, 1400),
  P        = c(25,   22.5, 20),
  ETc      = c(1200, 1320, 1440)
) |>
  mutate(Vdra_mm  = Irr + P - ETc,
         LF       = Vdra_mm/(Irr+P),
         EC_drain = 1.2/LF,
         Sel_kg_ha = Vdra_mm * 10 * EC_drain * f_EC_TDS,   # kg/ha/an
         LF_req   = 1.2/(5*2 - 1.2),
         OK       = LF >= LF_req)

print(scen, digits = 3)

ggplot(scen, aes(scenario, Sel_kg_ha, fill = OK)) +
  geom_col() +
  geom_hline(yintercept = 0) +
  labs(y = "Sel exporté (kg/ha·an)", x = NULL,
       title = "Export de sel par drainage selon scénarios CC") +
  theme_minimal()

Conclusion : dès CC+1, le drainage devient insuffisant : la salinité de la zone racinaire dépasse le seuil. Il faudrait alors augmenter l'irrigation, changer de culture (plus tolérante) ou installer un sub-drainage + re-circulation. Ce type d'analyse justifie l'approche ISAWIP fondée sur un bilan hydro-salin couplé.

Objectifs pédagogiques
  • Mettre en œuvre le bilan hydrique journalier d'une parcelle drainée.
  • Calculer et interpréter les indices SEW-30 et SDI à partir de séries temporelles de profondeur de nappe.
  • Visualiser les régimes hydriques pendant la saison de croissance à l'aide de ggplot2.
  • Relier les résultats aux seuils agronomiques (maïs, soya, prairie).
Prérequis R 
install.packages(c("tidyverse","lubridate","zoo"))
library(tidyverse)
library(lubridate)
library(zoo)
Rappel théorique

Le bilan hydrique journalier d'une parcelle drainée s'écrit :

\[ \Delta S = P - \mathrm{ETR} - R - Q_d \]

où \(P\) est la pluie, \(\mathrm{ETR}\) l'évapotranspiration réelle, \(R\) le ruissellement et \(Q_d\) le drainage souterrain (tous en mm/j).

L'indice SEW-30 cumule l'excès d'eau au-dessus de 30 cm de profondeur :

\[ \mathrm{SEW}_{30} = \sum_{i=1}^{n} \max\!\left(0,\;30 - h_i\right) \quad \text{[cm·jour]} \]

Le SDI (Stress Day Index) pondère chaque jour par une fonction de stress cultural \(CS_i\) :

\[ \mathrm{SDI} = \sum_{i=1}^{n} CS_i \cdot SD_i \]

Contexte du TP

Vous analysez les données d'une station météorologique et d'un réseau de piézomètres installés sur une parcelle de maïs de 15 ha à Saint-Hyacinthe, printemps-été 2024. Les observations couvrent 60 jours (1er mai au 29 juin).

Exercice 1 — Génération et visualisation des données 12 min

Générez un jeu de données synthétique représentatif :

set.seed(42)
n <- 60
dates <- seq.Date(as.Date("2024-05-01"), by = "day", length.out = n)

# Pluies (mm/j) — climat québécois printanier
P <- pmax(0, round(rgamma(n, shape = 0.35, rate = 0.08), 1))

# ETR (mm/j) suivant la courbe saisonnière
ETR <- 2 + 3*sin(2*pi*(1:n)/(2*n)) + rnorm(n, 0, 0.4)
ETR <- pmax(0.5, round(ETR, 1))

# Profondeur de nappe (cm) — plus la nappe est haute, plus h est petit
h <- 80 - 1.2*P + cumsum(rnorm(n, 0, 2.5))
h <- pmin(pmax(h, 5), 120) |> round(1)

meteo <- tibble(date = dates, P = P, ETR = ETR, h = h)
head(meteo)

Tâches :

  1. Calculez les statistiques descriptives (summary()) des trois variables.
  2. Produisez un graphique à deux axes : pluie (barres) et profondeur de nappe (ligne inversée).
  3. Repérez visuellement les 3 événements pluvieux majeurs.
Solution 
summary(meteo)

library(ggplot2)
coef <- max(meteo$h) / max(meteo$P)

ggplot(meteo, aes(x = date)) +
  geom_col(aes(y = P), fill = "#00549b", alpha = 0.7) +
  geom_line(aes(y = h/coef), color = "#961428", linewidth = 1) +
  scale_y_continuous(
    name = "Pluie (mm/j)",
    sec.axis = sec_axis(~ . * coef, name = "Profondeur de nappe (cm)")
  ) +
  labs(title = "Régime hydrique printanier — parcelle maïs",
       x = "Date") +
  theme_minimal()

# Les 3 plus fortes pluies
meteo |> slice_max(P, n = 3)

Lecture : on attend des remontées rapides de la nappe (h diminue) après les pluies les plus fortes, typiques d'un sol loameux avec bonne infiltration mais drainage latéral limité.

Exercice 2 — Bilan hydrique journalier 12 min

On suppose qu'il n'y a pas de ruissellement (R = 0) et que la variation de stock \(\Delta S\) peut être calculée par différences finies de la profondeur de nappe en utilisant une porosité de drainage \(f = 0{,}07\) (loam).

  1. Estimez \(Q_d\) (drainage en mm/j) à partir de \(Q_d = P - \mathrm{ETR} - \Delta S\).
  2. Calculez le cumul annuel de chaque poste.
  3. Vérifiez que \(\sum Q_d \ge 0\) et discutez les périodes de drainage actif.
Solution 
f <- 0.07            # porosité de drainage
meteo <- meteo |>
  mutate(
    # ΔS en mm/j : une baisse de h (nappe qui monte) = stock qui augmente
    dh   = c(0, diff(h)),                    # cm/j (positif = nappe descend)
    dS   = -dh * 10 * f,                     # mm/j (conversion cm→mm × f)
    Qd   = pmax(P - ETR - dS, 0)             # drainage positif seulement
  )

cumul <- meteo |>
  summarise(P = sum(P), ETR = sum(ETR), dS = sum(dS), Qd = sum(Qd))
print(cumul)

# Jours de drainage actif
meteo |> filter(Qd > 1) |> nrow()
meteo |> ggplot(aes(date, Qd)) +
  geom_col(fill = "#006642") +
  labs(title = "Drainage journalier estimé",
       y = "Qd (mm/j)", x = "Date") +
  theme_minimal()

Commentaire : sur 60 jours, on s'attend à une fraction drainée de l'ordre de 20–40 % de la pluie en sol loameux bien drainé. Les pics de Qd suivent directement les fortes pluies avec un retard de 0 à 2 jours.

Exercice 3 — Indice SEW-30 12 min

Le seuil de 30 cm correspond au haut de la zone racinaire des grandes cultures au Québec.

  1. Calculez le SEW-30 total de la série (cm·j).
  2. Calculez un SEW-30 glissant sur 7 jours pour identifier les périodes critiques.
  3. Comparez au seuil de tolérance pour le maïs : SEW-30 < 100 cm·j par saison.
Solution 
meteo <- meteo |>
  mutate(excess30 = pmax(0, 30 - h))

SEW30_total <- sum(meteo$excess30)
SEW30_total

# Moyenne glissante sur 7 jours
meteo <- meteo |>
  mutate(SEW7 = zoo::rollsum(excess30, k = 7, fill = NA, align = "right"))

ggplot(meteo, aes(date, SEW7)) +
  geom_line(color = "#961428", linewidth = 1) +
  geom_hline(yintercept = 14, linetype = "dashed", color = "grey50") +
  labs(title = "SEW-30 glissant (fenêtre 7 jours)",
       subtitle = paste("Cumul saison :", round(SEW30_total,1), "cm·j"),
       y = "SEW-30 (cm·j)", x = "Date") +
  theme_minimal()

if (SEW30_total < 100) {
  cat("Parcelle acceptable pour le maïs.\n")
} else {
  cat("Parcelle potentiellement risquée — revoir l'espacement.\n")
}

Lecture : un SEW-30 > 100 cm·j signifie une perte potentielle de 10–25 % du rendement selon le stade. Le SEW-30 glissant permet de cibler les 7-10 jours critiques pour une action correctrice (pompage, augmentation de capacité).

Exercice 4 — SDI pondéré par le stade phénologique 14 min

Le coefficient CS pondère le stress selon la sensibilité de la culture :

  • Semis (j 1–15) : CS = 0.6
  • Végétatif (j 16–35) : CS = 1.0
  • Floraison (j 36–50) : CS = 1.5 (stade le plus sensible)
  • Maturation (j 51–60) : CS = 0.7

Un jour est compté comme stress day si h < 30 cm, on prend alors SD = (30 - h)/30.

  1. Implémentez CS et SD.
  2. Calculez le SDI saisonnier.
  3. Estimez la perte de rendement relative selon \( Y/Y_0 = 1 - 0{,}004\cdot\mathrm{SDI} \).
Solution 
meteo <- meteo |>
  mutate(
    j  = row_number(),
    CS = case_when(
      j <= 15 ~ 0.6,
      j <= 35 ~ 1.0,
      j <= 50 ~ 1.5,
      TRUE    ~ 0.7
    ),
    SD  = ifelse(h < 30, (30 - h)/30, 0),
    SDI_d = CS * SD
  )

SDI <- sum(meteo$SDI_d)
cat("SDI saisonnier =", round(SDI, 2), "\n")

Y_rel <- 1 - 0.004 * SDI
cat("Rendement relatif estimé =", round(100*Y_rel, 1), "%\n")

meteo |>
  ggplot(aes(j, SDI_d, fill = CS)) +
  geom_col() +
  scale_fill_gradient(low = "#cfe3ff", high = "#961428") +
  labs(title = "Contribution journalière au SDI",
       x = "Jour de la saison", y = "CS · SD", fill = "CS") +
  theme_minimal()

Interprétation : un SDI > 25 correspond typiquement à une perte > 10 %. La pondération par CS met en évidence que les jours pluvieux pendant la floraison ont l'impact le plus sévère : c'est à cette période qu'il faut garantir la capacité de drainage.

Exercice 5 — Diagnostic et recommandation 10 min

Sur la base des indices calculés :

  1. Cette parcelle respecte-t-elle le seuil CPVQ (SEW-30 < 100 cm·j) pour le maïs ?
  2. Quels jours auraient requis une intensification du drainage (pompage) ?
  3. Proposez deux leviers d'amélioration (espacement, profondeur, drainage contrôlé).
Solution 
meteo |>
  filter(SD > 0.5) |>
  select(date, P, h, SD, CS, SDI_d) |>
  arrange(desc(SDI_d)) |>
  head(10)

diag <- tibble(
  indicateur = c("SEW-30 (cm·j)", "SDI", "Rendement relatif (%)"),
  valeur     = c(round(SEW30_total,1), round(SDI,2), round(100*Y_rel,1)),
  seuil      = c("< 100", "< 25", "> 90")
)
print(diag)

Pistes de discussion :

  • Si SEW-30 > 100 : réduire l'espacement L (ex. 18 m → 15 m) et vérifier q design.
  • Drainage contrôlé : abaisser la nappe en avril et la remonter après la floraison → gain azote + hydrique.
  • Analyse de tendance : refaire l'exercice pour 10 années consécutives pour qualifier un risque climatique.

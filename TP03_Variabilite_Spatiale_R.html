Exercice 1 : Analyse exploratoire de données spatiales

Durée : 25 minutes

Étape 1 : Création d'un jeu de données simulé

Nous allons créer 50 points de conductivité hydraulique (Ks en cm/h) sur une parcelle 500 m × 500 m à Meknès, avec une structure d'autocorrélation spatiale réaliste.

Afficher le code R
# Charger les bibliotheques
library(gstat)
library(sp)
library(ggplot2)

# Fixer la graine pour la reproductibilite
set.seed(42)

# Creer 50 points aleatoires dans le domaine [0, 500] x [0, 500]
n_points <- 50
x <- runif(n_points, 0, 500)
y <- runif(n_points, 0, 500)

# Generer Ks avec autocorrelation spatiale
g <- gstat(formula=z~1, locations=~x+y, dummy=TRUE, beta=5,
           model=vgm(psill=1.2, model="Exp", range=100, nugget=0.5),
           nmax=20)
ks_values <- predict(g, newdata=data.frame(x=x, y=y), nsim=1)$sim1

# Verifier que Ks > 0
ks_values <- abs(ks_values) + 2

# Creer un dataframe
data_ks <- data.frame(x=x, y=y, Ks=ks_values)
head(data_ks)
Résultat attendu
Vous devez obtenir un dataframe de 50 lignes avec des colonnes x, y et Ks. Les valeurs de Ks varient typiquement entre 1 et 6 cm/h pour un sol limoneux.

Étape 3 : Test de normalité

La normalité est importante pour certaines approches géostatistiques.

Afficher le code R
# Test de Shapiro-Wilk
test_norm <- shapiro.test(data_ks$Ks)
print(test_norm)

# Graphique Q-Q
qqnorm(data_ks$Ks, main="Graphique Q-Q normal")
qqline(data_ks$Ks, col=2)

# Si non-normal : transformation logarithmique
if (test_norm$p.value < 0.05) {
  cat("La distribution n'est pas normale (p =",
      round(test_norm$p.value, 4), ")\n")
  data_ks$logKs <- log(data_ks$Ks)
  test_norm_log <- shapiro.test(data_ks$logKs)
  cat("Apres transformation log : p =",
      round(test_norm_log$p.value, 4), "\n")
}
Question
La distribution de Ks est-elle normale ? Si non, pourquoi la transformation logarithmique pourrait-elle aider ? Comparez les p-values avant et après transformation.

Exercice 2 : Variogramme expérimental

Durée : 25 minutes

Étape 1 : Calcul du variogramme

Le variogramme expérimental mesure la dissemblance entre des valeurs séparées par une distance $h$ :

$$\gamma(h) = \frac{1}{2N(h)} \sum_{i=1}^{N(h)} [Z(x_i) - Z(x_i + h)]^2$$

où $N(h)$ est le nombre de paires à distance $h$.

Afficher le code R
# Creer un objet spatial pour gstat
coordinates(data_ks) <- ~x + y

# Calculer le variogramme experimental
vg_exp <- variogram(Ks ~ 1, data = data_ks)
print(vg_exp)

# Affichage graphique
plot(vg_exp, main="Variogramme experimental de Ks",
     xlab="Distance (m)", ylab="Semivariance (cm/h)^2")
Résultat attendu
Le variogramme affiche la semivariance en fonction de la distance. Vous devriez voir une courbe croissante qui se stabilise au-delà d'une certaine distance (la portée).

Étape 3 : Identification visuelle des paramètres

Identifiez les trois paramètres clés du variogramme.

Afficher le code R
# Affichage avec annotations
plot(vg_exp, main="Parametres du variogramme",
     xlab="Distance (m)", ylab="Semivariance (cm/h)^2")

# Pepite (nugget)
nugget <- vg_exp$gamma[1]
cat("Pepite estimee :", nugget, "\n")

# Palier (sill)
sill_est <- max(vg_exp$gamma)
cat("Palier estime :", sill_est, "\n")

# Portee (range)
range_est <- vg_exp$dist[which(vg_exp$gamma >= 0.95 * sill_est)[1]]
cat("Portee estimee :", range_est, "m\n")

# Ajouter des lignes de reference
abline(h = sill_est, col = "red", lty = 2, lwd = 2)
abline(v = range_est, col = "blue", lty = 2, lwd = 2)
Note
La pépite représente la variabilité à petite échelle et les erreurs de mesure. Le palier est la variance totale du processus. La portée est la distance au-delà de laquelle il n'y a plus de corrélation.

Étape 3 : Comparaison des modèles

Évaluez la qualité de l'ajustement pour chaque modèle.

Afficher le code R
# Fonction pour calculer SSE
sse <- function(vg_exp, vgm_fit) {
  vg_pred <- variogramLine(vgm_fit, dist_vector = vg_exp$dist)
  sum((vg_exp$gamma - vg_pred$gamma)^2)
}

# Calculer SSE pour chaque modele
sse_sph <- sse(vg_exp, vgm_sph)
sse_exp <- sse(vg_exp, vgm_exp)

cat("SSE modele spherique :", round(sse_sph, 4), "\n")
cat("SSE modele exponentiel :", round(sse_exp, 4), "\n")

# Affichage compare
plot(vg_exp, model = vgm_sph,
     main = "Comparaison des modeles",
     col = "black", pch = 20)
lines(variogramLine(vgm_sph, maxdist = max(vg_exp$dist),
                     n = 100), col = "blue", lwd = 2)
lines(variogramLine(vgm_exp, maxdist = max(vg_exp$dist),
                     n = 100), col = "red", lwd = 2)
legend("bottomright", c("Experimental", "Spherique", "Exponentiel"),
       pch = c(20, NA, NA), lty = c(NA, 1, 1),
       col = c("black", "blue", "red"))

Étape 4 : Validation croisée leave-one-out

Vérifiez la fiabilité du modèle ajusté.

Afficher le code R
# Validation croisee pour le modele spherique
cv_sph <- krige.cv(Ks ~ 1, data = data_ks, model = vgm_sph,
                    nfold = nrow(data_ks))

# Statistiques de validation
rmse_sph <- sqrt(mean((cv_sph$residual)^2))
me_sph <- mean(cv_sph$residual)
corr_sph <- cor(cv_sph$observed, cv_sph$var1.pred)

cat("Modele spherique :\n")
cat("  RMSE :", round(rmse_sph, 4), "\n")
cat("  ME :", round(me_sph, 4), "\n")
cat("  Correlation :", round(corr_sph, 4), "\n")

# Visualisation des residus
par(mfrow = c(1, 2))
plot(cv_sph$observed, cv_sph$var1.pred,
     main = "Observe vs Predit",
     xlab = "Ks observee", ylab = "Ks predite")
abline(0, 1, col = "red", lwd = 2)
hist(cv_sph$residual,
     main = "Distribution des residus",
     xlab = "Residus (cm/h)")
par(mfrow = c(1, 1))
Attention
Une validation croisée avec RMSE proche de 0, ME proche de 0 et corrélation proche de 1 indique un modèle fiable. Des déviations importantes signalent des biais ou une mauvaise spécification du modèle.
Question
Le modèle sphérique est-il fiable pour prédire Ks ? Observez-vous des résidus systématiques ? Que pourriez-vous faire pour améliorer le modèle ?
